Changelogs are the core of ReleaseDock. Each changelog entry belongs to a project and goes through a draft → published workflow.

## Editor

The changelog editor is a block-based rich text editor powered by BlockNote. It supports:

- Headings, paragraphs, and lists
- Inline formatting (bold, italic, code)
- Images with drag-and-drop upload
- Slash commands for quick block insertion
- Cover images

Changes are auto-saved as you type, so you never lose work.

## Statuses

Each changelog has one of three statuses:

| Status | Description |
|---|---|
| Draft | Work in progress. Not visible to users. |
| Scheduled | Will be published automatically at the scheduled date and time. |
| Published | Live and visible on the public page, widget, and email notifications. |

You can edit published changelogs at any time — changes are reflected immediately.

## Labels

Attach one or more labels to a changelog to categorize it. Labels are color-coded and appear on the public page and widget. Common examples:

- Feature
- Bugfix
- Improvement
- Breaking change

Create and manage labels in **Settings > Labels**.

## Cover images

Upload a cover image to make your changelog visually stand out. Cover images are displayed at the top of the changelog on both the public page and the widget.

## Public changelog page

Every project gets a public changelog page at `app.releasedock.co/changelog/your-slug`. This page lists all published changelogs in reverse chronological order. No login required for readers.

Users can react to changelogs with emoji reactions and view counts are tracked automatically.
