Labels and categories help you organize changelogs so readers can quickly find what's relevant to them.

## Labels

Labels are color-coded tags you attach to individual changelogs. They appear on the public changelog page and in the widget.

Common label examples:
- Feature (green)
- Bugfix (red)
- Improvement (blue)
- Breaking change (orange)

Create and manage labels in **Settings > Labels**. Each label has a name and a color.

## Categories

Categories are broader groupings for your changelogs. Subscribers can set category preferences to only receive email notifications for the categories they care about.

For example, you might have categories like "Product updates", "API changes", and "Security advisories" — and a developer subscriber might only opt into "API changes" and "Security advisories".

## Using labels and categories

When creating or editing a changelog, you can:

1. Attach one or more labels from the label picker
2. Assign categories to group the changelog

Labels are displayed inline on changelog entries. Categories are used behind the scenes for subscriber filtering and organization.
